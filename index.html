Project Lunaris

Role: Sole Programmer, Quest & Combat Designer
Team Size: 4
Status: Ongoing
Engine: Unreal Engine 5.4
Tech: C++, Blueprints


Overview

Project Lunaris is a turn-based RPG focused on player choice and systemic progression. As the sole programmer on the project, I was responsible for the implementation of all core gameplay systems, while also contributing directly to quest and combat design.


Systems & Responsibilities
  • Modular quest system
  • Branching dialogue system
  • Turn-based combat system
  • Elemental skill framework
  • Multiple gameplay and menu UI systems

Technical Highlights

Modular Quest Architecture:
I designed a flexible, step-based quest system where quests are composed of interchangeable steps that can be combined in different ways. This approach allows designers to build a wide variety of quest behaviors by assembling predefined step types, enabling rapid iteration and strong design freedom without requiring code changes.
Flag-Driven Dialogue System:
The dialogue system is built around a flag-based state model that allows NPCs to react dynamically to player actions, quest progress, and previous dialogue choices. Flags can be set by both quests and dialogues, enabling branching conversations and evolving narrative responses while keeping the system data-driven and designer-friendly.


Technical Challenge & Solution

To keep the dialogue flag system performant and readable, I used enums to represent dialogue states. However, Unreal Engine’s reflection system limits Blueprint-exposed enums to TEnumAsByte, which caps the number of usable values and reduced flexibility as the project grew. To solve this, I implemented a custom struct paired with an IPropertyTypeCustomization, allowing designers to work with a large, readable set of dialogue flags directly inside the editor while bypassing the 255-value limitation. This maintained runtime performance while significantly improving editor usability and long-term scalability.
